# lessons

- keep the context retrieval story concrete: api contract, graph walk, policy filtering, citations, and verifier proof should all line up.
- when explaining onboarding, lead with the real buyer/user journey and business outcome. control-plane endpoints are proof, not the product story.
- do not answer provenance/runtime questions with domain rag. questions like "are you legit?" or "are we using llms?" need direct runtime proof in the chat response itself.
- local llm proof is not enough. if the product is an agent chat, the final chat answer must be generated by the local model after tool/context calls, with deterministic code only as fallback.
